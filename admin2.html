02 — The Agent Swarm

How the Price Intelligence
System Works

The monitoring system uses three agent layers operating continuously. No autonomous purchasing — every action passes through Tony or Murr for final approval. The agents find the opportunity; the humans make the call.

/ Architecture / Real-Time Price & Arbitrage Monitor
▸ Agent 01 — The Watcher Monitors eBay Buy API every 15 min for specific catalog items. Monitors new Whatnot listings for vinyl in target genre clusters. Flags newly listed items matching Fred's catalog.
▸ Agent 02 — The Appraiser Compares eBay "Buy It Now" price vs. 30-day sales median for that specific pressing. Flags arbitrage opportunities where eBay price is >30% below market. Checks Fred's listed items for under-pricing.
▸ Agent 03 — The Alerter Sends Slack/SMS alert for arbitrage buys above $20 margin. Generates weekly "Reprice Report" for Murr + Tony. Queues mis-priced items for Tony's review in Shopify dashboard.
▸ Agent 04 — The Trend Spotter Monitors Reddit r/vinyl and Google Trends for albums spiking in cultural attention (artist tours, deaths, TikTok moments). Cross-references against current Fred's inventory. Alerts when action is needed.
↓ All agents report via webhook to a single Supabase dashboard ↓
Tony sees: "3 items to reprice. 1 arbitrage buy available. Fleetwood Mac trending — 2 copies in stock at $28, Discogs median: $65."
Important note: The Discogs API has commercial use restrictions on marketplace data. The architecture uses the eBay Developer API (legally licensed for commercial use) as the primary price benchmark, supplemented by Discogs for historical median reference via Fred's own authenticated seller account. This approach is fully compliant.
